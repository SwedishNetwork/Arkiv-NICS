Grundkonfiguration i Linux

Det finns ett enklare sätt att ställa in hostname, under installationen kan man nämnligen smidigt ange det hostname som senare kommer att användas. Behöver du däremot byta hostname därefter så är det tyvärr inte lika smidigt. Instruktionerna nedan guidar dig igenom hur man gör.

Observera att instruktionerna antar att du kan köra kommandon som root med hjälp utav sudo eller doas. Det vill säga att du har genomfört del 2 av guiden.

3. Hostname

Ett hostname är det namn som du döper din enhet till. Ditt hostname bör inte innehålla några punkter, då punkter senare används för beskriva den domän enheten tillhör.

Hostname

För att kontrollera nuvarande hostname kan du använda följande kommando (det står även ofta utskrivet i prompten).

user$ hostname

För att kontrollera nuvarande hostname kan du använda följande kommandon (det står även ofta utskrivet i prompten).

user$ doas nano /etc/hostname

För att kontrollera nuvarande hostname kan du använda följande kommandon (det står även ofta utskrivet i prompten).

user$ hostname
user$ hostnamectl

Det finns flera sätt att ändra hostname, enklast är dock att ersätta ditt tidigare hostname med ett nytt i tillhörande konfigurationsfil. 

user$ sudoedit /etc/hostname

Hosts

Om man byter hostname behöver man även justera de konfigurationsfiler som hänvisar till namnet. Den mest välkända konfigurationsfilen, och många gånger en av få filer som behöver uppdateras, är "/etc/hosts"-filen.

Beroende på vad man har installerat kan man dock även behöva uppdatera hostname på andra platser. Debians kommunitydrivna guide för att ändra hostname innehåller bland annat en lista över flera av dessa program: https://wiki.debian.org/Hostname#Rename_a_computer.

Beroende på om man använder DHCP/SLAAC eller statiska adresser kommer innehållet skilja sig lite åt. Oavsett vad som används så behöver du byta ut tidigare hostname med ditt nya.

user$ doas nano /etc/hosts

Om du använder dynamisk adressering och ditt hostname ska ändras till "linux" med FQDN "linux.home.arpa" bör filen se ut som nedan. Notera att antalet mellanslag/tabb mellan IP-adresser och hostnames inte spelar någon roll.

127.0.0.1       linux.home.arpa linux localhost.localdomain localhost
::1             localhost localhost.localdomain

Beroende på om man använder DHCP/SLAAC eller statiska adresser kommer innehållet skilja sig lite åt. Oavsett vad som används så behöver du byta ut tidigare hostname med ditt nya.

user$ sudoedit /etc/hosts

Om du använder dynamisk adressering och ditt hostname ska ändras till "linux" med FQDN "linux.home.arpa" bör filen se ut som nedan. Notera att antalet mellanslag/tabb mellan IP-adresser och hostnames inte spelar någon roll.

127.0.0.1       localhost
127.0.1.1       linux.home.arpa linux
::1             localhost ip6-localhost ip6-loopback
ff02::1         ip6-allnodes
ff02::2         ip6-allrouters

Om DHCP/SLAAC används behöver hosts filen inte uppdateras, med statisk adressering behöver man dock byta ut tidigare hostname med det nya.

user$ sudoedit /etc/hosts

Hosts filen bör se ut som nedan om man använder dynamisk adressering. Notera att antalet mellanslag/tabb mellan IP-adresser och hostnames inte spelar någon roll.

127.0.0.1       localhost

Applicera namnändringar

När hostname har uppdaterats överallt behöver man starta om för att applicera namnändringen.

user$ doas reboot
user$ sudo systemctl reboot
user$ sudo systemctl reboot
user$ sudo systemctl reboot

Efter omstarten kontrollera att hostname med FQDN applicerades korrekt.

user$ hostname -f

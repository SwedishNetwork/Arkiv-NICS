Grundkonfiguration i Linux

Instruktionerna guidar dig igenom hur man konfigurerar vanliga användarinställningar och tildelar högre behörighet åt en användare.

Observera att instruktionerna antar att du kan ansluta med SSH och logga in som root med hjälp utav substitute user. Det vill säga att du har genomfört del 1 av guiden.

2. Användarinställningar

Användarinställningar

I Linux sparas användarinställningar som environment variables. Dessa variabler kan antingen hanteras gemensamt för alla lokala användare, i filen "/etc/profile", eller per användare, i filen "~/.profile".

Ställ in en standard textredigerare och pager. Börja med att öppna "/etc/profile".

root# nano /etc/profile

Ange nano och less som standard textredigerare och pager respektive. Placera konfigurationsraderna högst upp i filen innan if-blocken.

export EDITOR=nano
export PAGER=less

Logga ut och logga in igen för att applicera inställningarna.

Root-behörighet

Redan under 60-talets utveckligen av TENEX såg man ett behov av att ge vanliga användare utökade rättigheter. Lösningen var något som kallades för wheel. TENEX lösning påverkade UNIX och det arvet syns fortfarande i flera BSD- och Linux-distributioner. Konventionellt används därför gruppen "wheel" för att ge användare rättighet att köra kommandon som root. Det förekommer dock även att andra grupper används.

Observera att exempeln nedan antar din användare heter "nics".

Alpine använder programmet "doas" för att ge vanliga användare möjligheten att köra kommandon som root (motsvarar ungefär Windows "Run as Administrator"). Installera doas och lägg till din användare i gruppen "wheel".

root# apk add doas
root# adduser nics wheel
root# nano /etc/doas.d/doas.conf

Öppna konfigurationsfilen och lägg till nedan konfigurationsrad.

permit persist keepenv :wheel

Debian använder programmet "sudo" för att ge vanliga användare möjligheten att köra kommandon som root (motsvarar ungefär Windows "Run as Administrator"). Installera sudo och lägg till din användare i gruppen "sudo".

root# apt install sudo
root# usermod -aG sudo nics
root# visudo

Inställning för sudo finns i filen "/etc/sudoers". Var dock observant när du editerar filen då minsta syntaxfel leder till att sudo slutar fungera. I en riktig driftsatt miljö får sådana misstag stora konsekvenser. Filen "/etc/sudoers" bör därför alltid editeras med kommandot "visudo". Kommandot öppnar "/etc/sudoers" med din standard textredigerare (som defineras av environment variablen "$EDITOR") och kontrollerar att inga syntaxfel har gjorts innan filen sparas.

Öppna konfigurationsfilen med kommandot "visudo" och se till att nedan konfigurationsrad finns. Notera att antalet mellanslag/tabb inte spelar någon roll.

%sudo	ALL=(ALL:ALL) ALL

Disable root

Eftersom du nu har en användare som kan köra kommandon som root (genom att lägga till "sudo" eller "doas" framför kommandot) behövs inte root-kontot längre. Root-användaren är en populär måltavla för attacker och det är därför ofta lämpligt att stänga av root-kontot när man har konfigurerat doas/sudo. I det här fallet görs det inte enbart av säkerhetsskäl utan också på grund utav pedagogiskaskäl.

Logga in som din användare och kontrollera att doas/sudo fungerar innan du stänger av root-kontot. Observera att doas/sudo frågar efter din användares lösenord och inte lösenordet till root-kontot

whoami
sudo whoami
doas whoami

Användare definieras i filen "/etc/passwd", bland annat lagras även information om användarens hemkatalog och skal. Precis som filnamnet antyder användes filen en gång i tiden även för lösenord, numera finns dock ingen känslig information i filen. Lösenord lagras istället i filen "/etc/shadow".

För att hindra användning av root-kontot ska du använda nedan kommandon. Det blockerar användning av lösenord och visar en varning vid inloggningsförsök

user$ doas sed -i 's/^root:/root:!/' /etc/shadow
user$ doas sed -i 's$root:x:0:0:root:/root:/bin/ash$root:x:0:0:root:/root:/sbin/nologin$' /etc/passwd

Debian kommer med ett par verktyg för att editera "/etc/passwd"- och "/etc/shadow"-filerna. Verktygen underlättar användarhantering då man slipper editera flera olika filer. Det minskar även risken för fel då filerna är ytterst känsliga. Vid minsta syntaxfel kan man lätt råka göra så att användaren inte längre kan logga in. Så länge man har bättre alternativ bör man med andra ord aldrig editera dessa filer manuellt.

Använd nedan kommando för att hindra användare från att logga in som root.

user$ sudo usermod -L root -s /sbin/nologin

Öppna manualen och kontrollera vad flaggorna ovan betyder. Avsluta sedan manualen med "q".

user$ man usermod
user$ man nologin

Men i och med att root-kontot används för att bland annat köra bakgrundsprocesser och schemalaggdajobb så är det inte önskvärt. Istället blockerar vi enbart interaktiva inloggningar.

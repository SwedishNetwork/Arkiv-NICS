Grundkonfiguration i Linux

Det finns ett enklare sätt att ställa in nästverksinställningar, under installationen kan man nämligen smidigt ange IP-adress, nätmak och gateway som senare kommer att användas. Behöver du däremot ändra nätverksinställningar efter installationen så är det tyvärr inte lika smidigt. Instruktionerna nedan guidar dig igenom hur man gör.

Observera att instruktionerna antar att du kan köra kommandon som root med hjälp utav sudo eller doas. Det vill säga att du har genomfört del 2 av guiden.

4. Network Settings

Interfaces

För att kunna ändra nätverksinställningar behöver man först ta reda på vilket eller vilka interface man har och vad de heter. Tidigare namngavs interface i den ordning de initierades vid uppstart, enligt den så kallade 'original simple scheme'. Interface namngavs då med ett löpnummer enligt: "eth0", "eth1" och så vidare. Denna namngivning har dock ersatts och idag används istället 'predictable names scheme', där interface namnges utifrån sin hårdvaruadress (t.ex. baserat på PCIe- eller USB-bussadress).

Även om 'original simple scheme' har ersatts finns det fortfarande fall där namngivningen inte har uppdaterats. Till exempel använder vissa virtuella nätverkskort fortfarande denna namnstandard, det blir dock alltmer ovanligt.

Du kan visa dina interface samt nuvarande nätverksinställningar med nedan kommandon.

user$ ifconfig
user$ route

Du kan visa dina interface samt nuvarande nätverksinställningar med nedan kommandon.

user$ ip address
user$ ip route

Du kan visa dina interface samt nuvarande nätverksinställningar med nedan kommandon.

user$ ip address
user$ ip route

Du kan visa dina interface samt nuvarande nätverksinställningar med nedan kommandon.

user$ nmcli
user$ ip address
user$ ip route

Du kan läsa mer om namnändringen här: https://www.freedesktop.org/wiki/Software/systemd/PredictableNetworkInterfaceNames/ och här: https://wiki.debian.org/NetworkInterfaceNames.

Nätverksinställningar

När du har tagit reda på vad ditt/dina nätverskinterface heter är du nu redo att gå vidare. Observera att exempeln nedan antar att det interface du konfigurerar heter "enp0s3".

Den officiella dokumentationen för Alpines nätverkskonfiguration hittar du här: https://wiki.alpinelinux.org/wiki/Configure_Networking. I manuallen finns det även ännu mer detaljerad information.

man interfaces
man resolv.conf

Instruktionerna nedan bör dock vara tillräckliga för de flesta scenarion.

/etc/network/interfaces

Alpine hanterar nätverksinställningar i filen "/etc/network/interfaces".

user$ doas nano /etc/network/interfaces

Innehållet i filen bör liknande nedan om du ska konfigurera statiskt adresserad dual-stack.

auto lo
iface lo inet loopback

auto enp0s3
iface enp0s3 inet static
  address 192.0.2.10/24
  gateway 192.0.2.1

iface enp0s3 inet6 static
  address 2001:db8:0:2::10/64
  gateway 2001:db8:0:2::1

För att applicera nätverksinställningarna behöver nätverkstjänsten startas om. Det är sedan lämpligt att kontrollera om inställningarna applicerades korrekt.

user$ doas rc-service networking restart
user$ rc-service networking status
user$ ifconfig
user$ route
/etc/resolv.conf

DNS-servrar och DNS-suffix konfigureras i filen "/etc/resolv.conf".

user$ doas nano /etc/resolv.conf

Gentoo har tydlig och koncis dokumentation för konfigurationsfilen: https://wiki.gentoo.org/wiki/Resolv.conf#Configuration.

När du anger ett DNS-suffix med "search" (samt eventuellt med "domain" för bakåtkompatabilitet) ska du välja ett lokalt domännamn (se RFC 8375 och RFC 6762 appendix G). Förutsatt att du inte har en existerande DNS-infrasturkur, i sådana fall bör domännamnet matcha denna.

search nics.lan
nameserver 1.1.1.1
nameserver 1.0.0.1
nameserver 2606:4700:4700::1111
nameserver 2606:4700:4700::1001
/etc/hosts

Om du använder statisk adresser behöver "/etc/hosts"-filen uppdateras med adressändringen. Filen behöver även uppdateras om du bytt DNS-suffix.

user$ doas nano /etc/hosts

Observera att exemplet nedan använder "linux" som hostname och "nics.lan" som DNS-suffix.

127.0.0.1        linux.nics.lan linux localhost.localdomain localhost
::1              localhost localhost.localdomain
192.0.2.10       linux.nics.lan linux
2001:db8:0:2::10 linux.nics.lan linux

Du kan dubbelkolla att ditt FQDN applicerades korrekt med nedan kommando.

user$ hostname -f

Den officiella dokumentationen för Debians nätverkskonfiguration hittar du här: https://wiki.debian.org/NetworkConfiguration. I manuallen finns det även ännu mer detaljerad information. 

man interfaces
man resolv.conf

/etc/network/interfaces

Debian hanterar IP-inställningar i filen "/etc/network/interfaces".

user$ sudoedit /etc/network/interfaces

Innehållet i filen bör liknande nedan om du ska konfigurera statiskt adresserad dual-stack.

auto lo
iface lo inet loopback

auto enp0s3
iface enp0s3 inet static
  address 192.0.2.10/24
  gateway 192.0.2.1

iface enp0s3 inet6 static
  address 2001:db8:0:2::10/64
  gateway 2001:db8:0:2::1

För att sedan applicera nätverksinställningarna behöver man ta bort existerande inställningar och sedan applicera de nya. Om du tidigare har använt dynamisk adressering kan "ifdown" eventuellt generera en varning om att ditt interface inte har någon konfiguration, vilket i sådana fall kan ignoreras.

user$ sudo ifdown enp0s3 && sudo ifup enp0s3
user$ ip address
user$ ip route
/etc/resolv.conf

DNS-servrar och DNS-suffix konfigureras i filen "/etc/resolv.conf".

user$ sudoedit /etc/resolv.conf

Gentoo har tydlig och koncis dokumentation för konfigurationsfilen: https://wiki.gentoo.org/wiki/Resolv.conf#Configuration.

När du anger ett DNS-suffix med "search" (samt eventuellt med "domain" för bakåtkompatabilitet) ska du välja ett lokalt domännamn (se RFC 8375 och RFC 6762 appendix G). Förutsatt att du inte har en existerande DNS-infrasturkur, i sådana fall bör domännamnet matcha denna.

search nics.lan
nameserver 1.1.1.1
nameserver 1.0.0.1
nameserver 2606:4700:4700::1111
nameserver 2606:4700:4700::1001
/etc/hosts

Om du använder statisk adresser behöver även "/etc/hosts"-filen uppdateras med adressändringen. Har du tidigare använt dynamiska adressering behöver du ersätta "127.0.1.1" med de statiska adresserna. Har du även bytt DNS-suffix behöver FQDN uppdateras.

user$ sudoedit /etc/hosts

Observera att exemplet nedan använder "linux" som hostname och "nics.lan" som DNS-suffix.

127.0.0.1        localhost
192.0.2.10       linux.nics.lan linux
2001:db8:0:2::10 linux.nics.lan linux
::1              localhost ip6-localhost ip6-loopback
ff02::1          ip6-allnodes
ff02::2          ip6-allrouters

Du kan dubbelkolla att ditt FQDN applicerades korrekt med nedan kommando.

user$ hostname -f

SUSE använder sig utav Wicked för att hantera nätverkskonfiguration från och med SLE 12 (samt motsvarande openSUSE versioner). Det tidigare ifcfg-formatet används dock fortfarnde för att konfigurera Wicked. Du kan läsa mer om nätverkshantering med Wicked här: https://documentation.suse.com/external-tree/en-us/sles/12-SP4/networking_with_wicked_in_suse_linux_enterprise_12_guide.pdf, samt allmänt om nätverkskonfiguration i SUSE här: https://documentation.suse.com/sles/15-SP1/html/SLES-all/cha-network.html#sec-network-manconf. I manuallen finns det även ännu mer detaljerad information.

user$ apropos wicked ifcfg netconfig
user$ man 5 ifcfg

Instruktionerna nedan bör dock vara tillräckliga för de flesta scenarion.

/etc/sysconfig/network/ifcfg-*

Varje interface har en ifcfg-konfigurationsfil under katalogen "/etc/sysconfig/network".

user$ sudoedit /etc/sysconfig/network/ifcfg-enp0s3

SUSE kommer med en detaljerad exempel-template för ifcfg-konfigurationsfiler.

user$ less /etc/sysconfig/network/ifcfg.template

Innehållet i filen bör dock liknande nedan om du ska konfigurera statiskt adresserad dual-stack.

STARTMODE=hotplug
BOOTPROTO=static
IPADDR1=192.0.2.10/24
IPADDR2=2001:db8:0:2::10/64
ZONE=public
/etc/sysconfig/network/routes

Statisk routing hanteras i en separat fil.

user$ sudoedit /etc/sysconfig/network/routes

Skapa filen om den inte finns. I filen ska routes till alla nätverk som är directly connected samt en default gateway defineras. Använd nedan konfiguationsutdrag som referens.

# Destionation          Gateway                 Deprecated      Interface
127.0.0.0/8             -                       -               lo
192.0.2.0/24            -                       -               enp0s3 
2001:db8:0:2::/64       -                       -               enp0s3 
default                 192.0.2.1               -               enp0s3
/etc/sysconfig/network/config

DNS-servrar hanteras utav netconfig (via Wicked) och konfigureras i den globala nätverkskonfigurationsfilen. Börja med att ta en backup-kopia på standardkonfigurationen utifall att du behöver återgå till denna.

user$ sudo cp /etc/sysconfig/network/config /etc/sysconfig/network/config.default
user$ sudoedit /etc/sysconfig/network/config

För att konfigurera DNS-inställningar behöver du justera nedan värden.

NETCONFIG_DNS_STATIC_SEARCHLIST="nics.lan"
NETCONFIG_DNS_STATIC_SERVERS="1.1.1.1 1.0.0.1 606:4700:4700::1111 2606:4700:4700::1001"
Applicera

För att sedan applicera nätverksinställningarna behöver man ta bort existerande inställningar och sedan applicera de nya. Observera att i nedan exempel konfigurerades interface "enp0s3".

user$ sudo wicked ifdown enp0s3 && sudo wicked ifup enp0s3
/etc/hosts

Om du använder statisk adresser behöver "/etc/hosts"-filen uppdateras med adressändringen. Filen behöver även uppdateras om du bytt DNS-suffix.

user$ sudoedit /etc/hosts

Observera att exemplet nedan använder "linux" som hostname och "nics.lan" som DNS-suffix.

192.0.2.10       linux.nics.lan linux
2001:db8:0:2::10 linux.nics.lan linux
127.0.0.1        localhost
::1              localhost ipv6-localhost ipv6-loopback
fe00::0          ipv6-localnet
ff00::0          ipv6-mcastprefix
ff02::1          ipv6-allnodes
ff02::2          ipv6-allrouters
ff02::3          ipv6-allhosts

Du kan dubbelkolla att ditt FQDN applicerades korrekt med nedan kommando.

user$ hostname -f

Red Hat använder sig utav Network Manager för att hantera nätverksinställningar. Tidigare har ifcfg-filer använts för nätverkskonfiguration men dessa har fasats ut med RHEL 9 (du kan läsa mer om detta här: https://www.redhat.com/en/blog/rhel-9-networking-say-goodbye-ifcfg-files-and-hello-keyfiles). Numera har varje interface en konfigurationsfil under katalogen "/etc/NetworkManager/system-connections" där både IP- och DNS-inställningar hanteras.

Den officiella dokumentation för nätverkskonfiguration med Network Manager är relativt teknisk då verktyget är både kraftfullt och komplext (https://networkmanager.dev/docs/api/latest/nm-settings-keyfile.html och https://networkmanager.dev/docs/api/latest/). För att se samtliga möjliga inställningar visas det enklast med nedan kommando (avsluta med "q").

nmcli connection show enp0s3

Instruktionerna nedan bör dock vara tillräckliga för de flesta scenarion.

/etc/NetworkManager/system-connections/

Börja med att ta en backup-kopia på standardkonfigurationen utifall att du behöver återgå till denna. Observera att exemplen nedan konfigurerar "enp0s3", ditt interface kan mycket väl heta något annat.

user$ sudo cp /etc/NetworkManager/system-connections/enp0s3.nmconnection /etc/NetworkManager/system-connections/enp0s3.nmconnection.default 
user$ sudoedit /etc/NetworkManager/system-connections/enp0s3.nmconnection

För att konfigurerar statiskt adresserad dual-stack, samt DNS-servrar och DNS-suffix bör innehållet i filen likna nedan. Standardkonfigurationen bör annars behållas som den är.

När du anger ett DNS-suffix med "dns-search" ska du välja ett lokalt domännamn (se RFC 8375 och RFC 6762 appendix G). Förutsatt att du inte har en existerande DNS-infrasturkur, i sådana fall bör domännamnet matcha denna.

[ipv4]
method=manual
address1=192.0.2.10/24
gateway=192.0.2.1
dns=1.1.1.1;1.0.0.1
dns-search="nics.lan"

[ipv6]
method=manual
address1=2001:db8:0:2::10/64
gateway=2001:db8:0:2::1
dns=2606:4700:4700::1111;2606:4700:4700::1001
dns-search="nics.lan"
nmcli

För att applicera konfigurationen behöver du be Network Manager att läsa in konfigurationsfilerna på nytt, samt stänga ner och starta upp berörda interface.

user$ sudo nmcli connection reload
user$ sudo nmcli connection down enp0s3 && sudo nmcli connection up enp0s3
user$ nmcli
user$ ip address
user$ ip route
/etc/hosts

Om du använder statisk adresser behöver "/etc/hosts"-filen uppdateras med adressändringen. Filen behöver även uppdateras om du bytt DNS-suffix.

user$ sudoedit /etc/hosts

Observera att exemplet nedan använder "linux" som hostname och "nics.lan" som DNS-suffix.

127.0.0.1        localhost localhost.localdomain localhost4 localhost4.localdomain4
::1              localhost localhost.localdomain localhost6 localhost6.localdomain6
192.0.2.10       linux.nics.lan linux
2001:db8:0:2::10 linux.nics.lan linux

Du kan dubbelkolla att ditt FQDN applicerades korrekt med nedan kommando.

user$ hostname -f

user$ ping google.com
